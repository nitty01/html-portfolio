QLM (Queue Length Management)

S.O.L.I.D. Design Principles

  • Single Responsibility: Computer vision, alerting, and configuration logic are separated into distinct modules for clarity and maintainability.
  • Open/Closed Principle: New detection models and alerting strategies can be added without changing the core queue management logic.
  • Liskov Substitution: All queue detection models implement a shared interface, so models can be swapped or upgraded seamlessly.
  • Interface Segregation: APIs for camera ingestion, admin configuration, and alerting are kept distinct, so clients only use what they need.
  • Dependency Inversion: Alerting and notification systems are abstracted, allowing for easy integration of new channels or services.
