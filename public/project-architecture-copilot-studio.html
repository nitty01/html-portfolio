Platform Capabilities

An interactive studio-not a one-shot chat. Architects work in copilot conversation and a live canvas while the platform advances through governed design stages backed by LangGraph and Azure Functions.

Copilot Chat

Stage 0

AI-guided dialogue across the engagement-context, constraints, and stage transitions tied to the active design step.

Business Value: Natural interaction without losing structured pipeline progress.

Diagram Canvas

Workspace

Real-time Mermaid rendering and editing; SignalR syncs canvas state so architects override or refine AI output instantly.

Business Value: User control over the architecture artifact, not read-only AI diagrams.

Requirements (AI)

Stage ①

Structured requirement synthesis-scope, NFRs, constraints-before any diagram is generated.

Business Value: Grounds downstream diagrams and cost models in agreed needs.

Cloud-Agnostic Mermaid

Stage ②

L1/L2/L3 diagrams as vendor-neutral Mermaid; persisted to Blob and editable on the canvas.

Business Value: One logical architecture before locking into a cloud.

Cloud Mapping

Stage ③

Maps each component in the Mermaid graph to concrete Azure, AWS, or GCP services; re-runs when the canvas changes.

Business Value: Target-cloud blueprints derived from the same neutral design.

Cost Estimation

Stage ④

Monthly and yearly estimates per mapped stack; Cost Gate blocks runaway model/tool spend before execution.

Business Value: FinOps visibility at blueprint time, not after deployment.

Multi-Cloud Compare

Stage ⑤

Side-by-side comparison of Azure, AWS, and GCP options for every component used in the architecture-cost, fit, and operational notes.

Business Value: Informed cloud choice from the actual component graph, not generic benchmarks.

Key Components

Studio shell: Copilot chat, diagram canvas, stage stepper-Front Door, API Management, Entra ID

Real-time canvas: Azure SignalR for live Mermaid edits, stage status, and collaborative session sync

LangGraph stages: Requirements → Mermaid → cloud map → monthly/yearly cost → multi-cloud compare → publish

Artifact store: Cosmos DB session/checkpoints; Blob for Mermaid source, mappings, estimates, compare reports

Stage tools (Functions): synthesize requirements, generate Mermaid, map clouds, estimate periods, compare matrix

AI + FinOps: Azure OpenAI gateway, AI Search pricing refs, Cost Gate and Cost Management chargeback

Architecture Decisions

  • Cloud-agnostic Mermaid before cloud mapping-neutral design drives Azure/AWS/GCP targets and comparisons
  • Canvas edit loops re-trigger mapping and cost stages without restarting the full engagement session
  • Staged LangGraph pipeline with human gates-each stage persisted and resumable via Cosmos DB and Service Bus
  • FinOps Cost Gate before heavy model calls; monthly and yearly estimates tied to mapped components

Architecture Copilot Studio

GenAI Platform | Global Consulting Firm

Business Problem

Architects needed a single workspace to gather requirements with AI, produce editable architecture diagrams, map those designs to target clouds, understand monthly and yearly cost, and compare Azure, AWS, and GCP options-without juggling separate tools or losing control of the diagram.

Strategic Decision

Build an interactive studio: copilot conversation plus a real-time diagram canvas, with a staged LangGraph backend that advances only when each step (requirements, Mermaid, cloud mapping, cost, comparison) is complete and architect-approved.

Architecture at a Glance

On Azure, architects use copilot chat and a SignalR-backed Mermaid canvas. LangGraph on Container Apps runs five AI stages; Functions implement stage tools; Cosmos DB and Blob hold session, canvas, mappings, and cost/compare outputs; Cost Gate and Azure OpenAI power governed, FinOps-aware generation.

Key Trade-offs

  • Neutral Mermaid first: Cloud mapping and cost/comparison derive from one editable graph-not three separate diagrams.
  • Canvas-driven loops: User edits on the canvas re-invoke map/cost stages without restarting requirements from scratch.
  • Staged gates: Human approval between stages; FinOps Cost Gate before expensive model/tool chains.

Business Outcome

Golden-path CI failures dropped from roughly one in four to under one in ten. Architects regained hours per engagement via checkpoint resume instead of manual state copy-paste. Material reduction in monthly Azure OpenAI spend after mandatory design-time gates. Platform patterns reused for adjacent global programs.
