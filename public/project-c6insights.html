Technical Architecture

  • Data Ingestion: Streaming vehicle data via Kafka → PySpark (AWS EMR) → S3 & MongoDB; Fuel Price Scraper (Lambda); AQI via API; Monthly manual data (Electricity Prices, Grid Emissions, Replacement Vehicles)
  • Storage: MongoDB (raw data), PostgreSQL (job metadata), S3 (data lake), Chroma DB (EV incentives vector store for RAG)
  • Processing Platform: Deep View Analytics - Master-slave architecture, horizontally scalable, batch-based processing with RabbitMQ for job parallelization (delay queue & DLQ)
  • GenAI Pipeline: Ollama LLM with RAG (Chroma DB) for EV incentives and MCP for vehicle data access
  • GenAI Use Cases: Vehicle-level replacement identification (cost, fuel, carbon, travel/service area) and fleet transformation roadmap generation (year-on-year with ROI & carbon savings)
  • API Layer: FastAPI for data serving, metadata enhancement, and simulation endpoints
  • Simulation Flow: UI → FastAPI → LLM processing → Vehicle replacement → Fleet roadmap → API response → RabbitMQ completion notification
  • Infrastructure: Kubernetes pod-based processing, Docker containers, AWS Cloud services
